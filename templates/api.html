Summary

This overview will give you some idea of what data is available, how to find what you need, and what you will see when you make a request to us.

The easiest way to see what data is included is to explore the data. You can build complex queries quickly and browse the results.

Please ensure that you read the guide for polling applications below before starting.

Results are returned in JSON format.

Endpoints

We provide several endpoints to retrieve different items:

  • All Documents
  • Document Items
  • Document Attributes
  • Authors
  • Regions
  • Instruments
  • Sampling Rates

For each endpoint:

  • Results can be filtered using parameters
  • Response contains minimal detail by default but more data can be exposed using parameters

Available Parameters case-sensitive

Below are the exposed keys.

  • pi
  • firstName
  • region
  • date
  • instrument
  • samplingRate
  • dateSort
  • all

Parameters Explained case-sensitive

pi

Last name of publisher, beginning with capital letter

firstName

First name of publishers, beginning with capital letter

region

Regional geographic location. Each word seperated by space, and each word begins with capital letter, i.e: Pacific Ocean

date

Date published. Format, brackets exclusive: [year][month][day], padded with zeroes, no extra characters, i.e: 20180930

instrument

Sensor instrument used, beginning with capital letter

samplingRate

Rate of sampling, in numerical format, i.e: 800

dateSort

Return results date sorted from recent to old

all

Returns all documents in collection

Get All Documents

Description Get everything in a collection
Endpoint /query/metadata?all
HTTP Method GET
Example: https://hwss.azurewebsites.net/query/metadata?all

Get Document Attributes

Description Get documents keys
Endpoint /query/metadata?categories
HTTP Method GET
Example: https://hwss.azurewebsites.net/query/metadata?categories

Multiparameter Query

Description Filter using multiple parameters to get more specific results
Endpoint /query/metadata?KEY={}&KEY2={}&KEY3{}
HTTP Method GET
Example: https://hwss.azurewebsites.net/query/metadata?firstName=Conard,Lisa®ion=Southern%20Ocean,Atlantic%20Ocean

Sort by Ascending Date

Description Get everything in collection and sort from oldest to newest.
Endpoint /query/metadata?all&dateSort=old
HTTP Method GET
Example: https://hwss.azurewebsites.net/query/metadata?all&dateSort=old

Sort by Descending Date

Description Get everything in collection and sort from newest to oldest.
Endpoint /query/metadata?all&dateSort=new
HTTP Method GET
Example: https://hwss.azurewebsites.net/query/metadata?all&dateSort=new

Get a Publisher

Description Get a specific publisher
Endpoint /query/metadata?pi={lastName}&firstName={firstName}
HTTP Method GET
Example: /query/metadata?pi=Faraon&firstName=Conard
Example: https://hwss.azurewebsites.net/query/metadata?pi=Faraon&firstName=Conard

Get Several Publishers

Description Specify more than one publisher, seperated by comma. Last names are seperated from first names. When listing, keep them in order to match last name in list to first name in list
Endpoint /query/metadata?pi={lastName1},{lastName2}&firstName={firstName1},{firstName2}
HTTP Method GET
Example: https://hwss.azurewebsites.net/query/metadata?pi=Faraon,Kim&firstName=Conard,Lisa

Get All Uniq Publishers

Description Get all list of uniq publishers
Endpoint /query/metadata?uniqNames
HTTP Method GET
Example: https://hwss.azurewebsites.net/query/metadata?uniqNames

Get a Region

Description Get a specific geographic location. Each word begins with captial letter, seperated by single space.
Endpoint /query/metadata?region={region Name}
